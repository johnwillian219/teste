Particionamento e Formatação de disco

O particionamento e formatação de discos são etapas fundamentais na preparação de um disco para armazenamento de dados ou instalação de sistemas operacionais. Vamos abordar o tema em partes, cobrindo os conceitos, tipos de particionamento, sistemas de arquivos, ferramentas utilizadas e procedimentos práticos.

  2. Conceitos Básicos

    • Particionamento: Dividir um disco físico em seções lógicas (partições) para organizar dados ou instalar diferentes sistemas operacionais.
    • Formatação: Preparar uma partição para armazenar dados, aplicando um sistema de arquivos que gerencia como os dados serão gravados e lidos.

  3. Tipos de Particionamento

    1. MBR (Master Boot Record)
      2.
      • Suporta até 4 partições primárias ou 3 primárias + 1 estendida.
      • Tamanho máximo do disco: 2 TB.
      • Compatível com sistemas mais antigos.
    2. GPT (GUID Partition Table)
      3.
      • Suporta discos maiores que 2 TB.
      • Permite até 128 partições primárias.
      • Mais confiável, com backups redundantes da tabela de partição.

    Para verificar se o seu disco é GPT ou MBR, abra o Prompt de Comando como administrador e execute o comando:

    diskpart

    Dentro do diskpart, liste os discos:

    list disk

    Verifique a coluna GPT, se houver um asterisco (*) na coluna GPT, o disco é GPT, caso contrário, é MBR.

  4. Tipos de Partições

    • Primária: Contém dados ou sistemas operacionais diretamente.
    • Estendida: Uma partição que contém subpartições chamadas partições lógicas.
    • Lógica: Subdivisões dentro de uma partição estendida.

    O identificador mais fácil para identificar se uma partição é primária ou estendida é verificar se a partição é inicializável ou não. Se a resposta for sim, é uma partição primária.

  5. Sistemas de Arquivos

    6. Um sistema de arquivos organiza e gerencia como os dados são armazenados e acessados em dispositivos de armazenamento, como discos e pendrives. Ele permite criar, ler e excluir arquivos, além de definir regras para organização, permissões e uso do espaço. Exemplos:

    • FAT32: Compatível com sistemas antigos, mas suporta arquivos de no máximo 4 GB.
    • NTFS: Padrão em Windows, suporta arquivos grandes e recursos como permissões e criptografia.
    • exFAT: Ideal para pendrives e dispositivos móveis, compatível com arquivos grandes.
    • ext4: Padrão em sistemas Linux, eficiente e confiável.
    • APFS: Sistema de arquivos da Apple, otimizado para SSDs.
